Case Study for Movie App Angular

Overview:

This is a database Built App build with Angular. The main point of the app is to display a database of movies and allow users to interact with them, by adding them to favorite list.

Purpose & Context:

myFlix-Angular was a personal project I build as part of my web development course at CareerFoundry to prove my mastery of full-stack JavaScript development.

Objective:

The aim was to have a full-stack project, I can add to my portfolio. The problem was to build the complete server-side and client-side for the application.

Server-Side:

I created an API using Node and Express. To retrieve data from and store data in the database, CRUD methods are used. To test the API, I used Postman, I also included JWT authentication and basic HTTP authentication.



Client-Side:

After being done with the API, I started to build the interface with Angular. It provides several interface views, a main view, a login view, a registration view and a profile view.

Challenges:

The challenges were to build the API. Because it was the first time building one, but managed to understand it quick. Developing the client side, wasn’t that hard, but it took me a little to understand it all and, actually it went better than I expected. But one thing that I want to change is being more organized with my code. There were times where I didn’t know how to do an exercise (Favorite Button). With the help from Google and also from my Tutor an Mentor, I managed to learn something new and also finish the task.

Duration:

The development took a lot more than I expected, because there were some problems with my code. In the end, I managed to find the bugs and fix them.
